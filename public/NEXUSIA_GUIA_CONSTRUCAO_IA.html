Engenharia de Prompts — O Segredo da "Inteligência"

A "inteligência" de uma IA não vem do modelo em si, mas de COMO você instrui o modelo. Isso se chama Prompt Engineering.

2.1 System Prompt Estruturado

O system prompt é a instrução invisível que define o comportamento da IA. Enviado antes de cada conversa, o usuário nunca vê.

Você é um assistente de inteligência artificial avançado da plataforma NexusIA.

MODOS DE OPERAÇÃO:
1. CONVERSACIONAL — Responda perguntas, explique conceitos, forneça conselhos
2. EXECUÇÃO — Realize tarefas, divida em etapas, execute planos
3. ANÁLISE — Analise dados, documentos, problemas e cenários
4. AUTOMAÇÃO — Crie fluxos automáticos, sugira integrações

RACIOCÍNIO ESTRUTURADO para tarefas complexas:
1. Interprete o pedido
2. Divida em subtarefas
3. Crie um plano
4. Execute cada etapa
5. Verifique resultados
6. Entregue a solução

ESTILO: claro, natural, profissional. Use markdown para formatação.
Seja proativo — sugira melhorias e alternativas quando relevante.

IMPORTANTE: Nunca se apresente, nunca diga quem você é,
nunca faça introduções sobre si mesmo. Vá direto ao ponto.

2.2 Técnicas Aplicadas

a) Modos de Operação: Em vez de "seja útil", liste modos específicos. Isso dá ao modelo um framework mental.

b) Chain-of-Thought: Instrua o modelo a seguir passos: interpretar → dividir → planejar → executar → verificar → entregar.

c) Estilo e Formatação: Diga para usar markdown, listas, tabelas, blocos de código.

d) Proatividade: "Sugerir melhorias e alternativas" faz a IA parecer mais inteligente.

e) Restrições: "Nunca se apresente" elimina respostas genéricas.

2.3 Erros Comuns

"Seja um assistente útil" → Genérico demais
Listar modos de operação + raciocínio estruturado
Prompts longos demais com regras contraditórias
Prompts concisos, organizados em seções claras

Como Funciona

const agentSystemPrompts = {
  research: `Você é um assistente especializado em pesquisa...
    IMPORTANTE: Nunca se apresente. Vá direto ao ponto.`,

  coder: `Você é um assistente especializado em programação...
    IMPORTANTE: Nunca se apresente. Vá direto ao ponto.`,

  // ... outros agentes
};

// Seleção dinâmica:
const systemPrompt = agent && agentSystemPrompts[agent]
  ? agentSystemPrompts[agent]
  : defaultSystemPrompt;
Prompts especializados > Prompts genéricos. O modelo "entra no personagem" e as respostas ficam mais focadas.

Streaming em Tempo Real (SSE)

Em vez de esperar a resposta completa, mostramos token por token em tempo real.

Backend (Edge Function)

const response = await fetch("https://ai-gateway/v1/chat/completions", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "google/gemini-3-flash-preview",
    messages: [{ role: "system", content: systemPrompt }, ...messages],
    stream: true, // ← ATIVA O STREAMING
  }),
});

return new Response(response.body, {
  headers: { "Content-Type": "text/event-stream" },
});

Frontend (Parser SSE)

const reader = resp.body.getReader();
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });

  while ((newlineIndex = buffer.indexOf('\n')) !== -1) {
    let line = buffer.slice(0, newlineIndex);
    buffer = buffer.slice(newlineIndex + 1);

    if (!line.startsWith('data: ')) continue;
    const jsonStr = line.slice(6).trim();
    if (jsonStr === '[DONE]') break;

    const parsed = JSON.parse(jsonStr);
    const content = parsed.choices?.[0]?.delta?.content;
    if (content) onDelta(content); // ← Token aparece na tela
  }
}

Armadilhas do SSE

NÃO dividir por "\n\n" — JSON pode quebrar entre chunks
NÃO criar nova mensagem a cada token — atualizar a existente
SEMPRE tratar JSON incompleto — re-bufferizar e esperar mais dados
SEMPRE tratar [DONE], CRLF (\r\n) e linhas de comentário (:)

Gerenciamento de Estado

Zustand — mais simples que Redux, sem boilerplate.

import { create } from 'zustand';

const useChatStore = create((set, get) => ({
  conversations: [],
  activeConversationId: null,
  selectedAgent: null,
  sidebarOpen: true,
  theme: 'dark',

  createConversation: (agent) => {
    const id = generateId();
    set((state) => ({
      conversations: [{ id, title: 'Nova conversa', messages: [], agent },
                       ...state.conversations],
      activeConversationId: id,
    }));
    return id;
  },

  addMessage: (conversationId, message) => {
    set((state) => ({
      conversations: state.conversations.map((c) => {
        if (c.id !== conversationId) return c;
        const messages = [...c.messages, { ...message, id: generateId() }];
        const title = c.messages.length === 0 && message.role === 'user'
          ? message.content.slice(0, 40) + '...'
          : c.title;
        return { ...c, messages, title };
      }),
    }));
  },
}));

Tratamento de Erros

// Backend — Rate Limit (429)
if (response.status === 429) {
  return new Response(
    JSON.stringify({ error: "Muitas requisições. Aguarde." }),
    { status: 429 }
  );
}

// Backend — Créditos (402)
if (response.status === 402) {
  return new Response(
    JSON.stringify({ error: "Créditos insuficientes." }),
    { status: 402 }
  );
}

// Frontend
try {
  await streamChat({ messages, onDelta, onDone, onError: (error) => {
    toast.error(error);
  }});
} catch (e) {
  toast.error('Erro ao conectar com a IA. Tente novamente.');
}

Dicas e Boas Práticas

Envie SEMPRE o histórico completo da conversa (para contexto)
Use system prompts específicos por agente/funcionalidade
Instrua o modelo a NÃO se apresentar
Use streaming para UX responsiva
Trate TODOS os erros (429, 402, 500, network)
Renderize markdown nas respostas da IA
Auto-renomeie conversas com base na primeira mensagem
Nunca exponha API keys no frontend
Nunca chame a IA diretamente do navegador
Nunca crie nova mensagem a cada token (atualize a existente)
Nunca use prompts genéricos como "seja útil"
Nunca ignore erros de rate limit

🧠 Resumo: O Que Faz a IA Parecer "Inteligente"

  1. Prompt Engineering — O fator #1. Defina modos de operação, raciocínio estruturado, estilo e restrições claras.
  2. Especialização — Agentes com prompts focados respondem melhor do que um prompt genérico.
  3. Streaming — Mostrar a resposta token por token dá sensação de velocidade e inteligência.
  4. Markdown Rico — Respostas formatadas com código, listas e tabelas parecem mais profissionais.
  5. Contexto — Enviar o histórico completo mantém a coerência na conversa.
  6. UX Polida — Animações, scroll suave, indicadores de loading.
  7. Restrições — Dizer o que a IA NÃO deve fazer é tão importante quanto o que ela deve fazer.
