================================================================================
PROMPT PARA RECRIAR O PROJETO NEXUSIA (SYNAPSE AI) — PLATAFORMA DE CHAT COM IA
================================================================================

Crie uma aplicação web completa de chat com inteligência artificial chamada "NexusIA" (exibida como "SYNAPSE AI" na tela inicial). A aplicação é uma plataforma de assistente IA com 7 agentes especializados, geração de imagens por IA, streaming de respostas token-a-token, tema claro/escuro, e interface moderna tipo ChatGPT.

================================================================================
STACK TECNOLÓGICA
================================================================================

- React 18 + TypeScript + Vite
- Tailwind CSS 3 com design system customizado (tokens HSL)
- shadcn/ui para componentes base (Dialog, Toast, etc.)
- Zustand para gerenciamento de estado
- Framer Motion para animações
- react-markdown para renderizar respostas em markdown
- react-router-dom para rotas
- lucide-react para ícones
- Sonner para notificações toast
- @tanstack/react-query
- Fontes: Inter (display) + JetBrains Mono (código)

Backend:
- Supabase Edge Functions (Deno) para proxy da API de IA
- API OpenAI-compatible (qualquer gateway compatível, ex: OpenAI, Gemini, etc.)
- Streaming via SSE (Server-Sent Events)

ZUSTAND STORE (src/store/chatStore.ts)
================================================================================

O store gerencia:
- Lista de conversas (conversations: Conversation[])
- Conversa ativa (activeConversationId)
- Agente selecionado (selectedAgent)
- Estado da sidebar (sidebarOpen)
- Tema (theme: 'light' | 'dark') — salvo em localStorage com key 'nexusia-theme'

Funcionalidades:
- createConversation(agent?) — cria nova conversa, opcionalmente com agente
- setActiveConversation(id) — seleciona conversa
- addMessage(conversationId, message) — adiciona mensagem; auto-renomeia conversa com os primeiros 40 chars da 1ª mensagem do usuário
- deleteConversation(id) — remove conversa
- renameConversation(id, title) — renomeia conversa
- getActiveConversation() — retorna conversa ativa
- setTheme(theme) — alterna tema e persiste em localStorage, togglea classe 'dark' no documentElement

IDs são gerados com: Math.random().toString(36).substring(2, 15)

```typescript
// Código completo do store:
import { create } from 'zustand';
import { AgentType, Conversation, Message } from '@/types/agent';

interface ChatState {
  conversations: Conversation[];
  activeConversationId: string | null;
  selectedAgent: AgentType | null;
  sidebarOpen: boolean;
  theme: 'light' | 'dark';
  setSelectedAgent: (agent: AgentType | null) => void;
  setSidebarOpen: (open: boolean) => void;
  setTheme: (theme: 'light' | 'dark') => void;
  createConversation: (agent?: AgentType) => string;
  setActiveConversation: (id: string | null) => void;
  addMessage: (conversationId: string, message: Omit<Message, 'id' | 'timestamp'>) => void;
  getActiveConversation: () => Conversation | undefined;
  deleteConversation: (id: string) => void;
  renameConversation: (id: string, title: string) => void;
}

const generateId = () => Math.random().toString(36).substring(2, 15);

const savedTheme = (typeof window !== 'undefined' && localStorage.getItem('nexusia-theme') as 'light' | 'dark') || 'dark';
if (typeof window !== 'undefined') {
  document.documentElement.classList.toggle('dark', savedTheme === 'dark');
}

export const useChatStore = create<ChatState>((set, get) => ({
  conversations: [],
  activeConversationId: null,
  selectedAgent: null,
  sidebarOpen: true,
  theme: savedTheme,

  setSelectedAgent: (agent) => set({ selectedAgent: agent }),
  setSidebarOpen: (open) => set({ sidebarOpen: open }),

  setTheme: (theme) => {
    localStorage.setItem('nexusia-theme', theme);
    document.documentElement.classList.toggle('dark', theme === 'dark');
    set({ theme });
  },

  createConversation: (agent) => {
    const id = generateId();
    const conversation: Conversation = {
      id, title: 'Nova conversa', messages: [], agent, createdAt: new Date(), updatedAt: new Date(),
    };
    set((state) => ({
      conversations: [conversation, ...state.conversations],
      activeConversationId: id,
      selectedAgent: agent || state.selectedAgent,
    }));
    return id;
  },

  setActiveConversation: (id) => set({ activeConversationId: id }),

  addMessage: (conversationId, message) => {
    const msg: Message = { ...message, id: generateId(), timestamp: new Date() };
    set((state) => ({
      conversations: state.conversations.map((c) => {
        if (c.id !== conversationId) return c;
        const messages = [...c.messages, msg];
        const title = c.messages.length === 0 && message.role === 'user'
          ? message.content.slice(0, 40) + (message.content.length > 40 ? '...' : '')
          : c.title;
        return { ...c, messages, title, updatedAt: new Date() };
      }),
    }));
  },

  deleteConversation: (id) => {
    set((state) => {
      const filtered = state.conversations.filter((c) => c.id !== id);
      return {
        conversations: filtered,
        activeConversationId: state.activeConversationId === id ? null : state.activeConversationId,
      };
    });
  },

  renameConversation: (id, title) => {
    set((state) => ({
      conversations: state.conversations.map((c) => c.id === id ? { ...c, title } : c),
    }));
  },

  getActiveConversation: () => {
    const { conversations, activeConversationId } = get();
    return conversations.find((c) => c.id === activeConversationId);
  },
}));
```

STREAMING CHAT CLIENT (src/lib/streamChat.ts)
================================================================================

Faz fetch POST para a edge function de chat, lê streaming SSE linha por linha, e emite tokens via callback onDelta. Trata erros 429 (rate limit) e 402 (sem créditos).

```typescript
import { AgentType } from '@/types/agent';

type Msg = { role: 'user' | 'assistant' | 'system'; content: string };

// A URL deve apontar para seu backend de chat (edge function, API route, etc.)
const CHAT_URL = `${YOUR_BACKEND_URL}/functions/v1/chat`;

export async function streamChat({
  messages, agent, onDelta, onDone, onError,
}: {
  messages: Msg[];
  agent?: AgentType;
  onDelta: (text: string) => void;
  onDone: () => void;
  onError: (error: string) => void;
}) {
  const resp = await fetch(CHAT_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${YOUR_API_KEY}`,
    },
    body: JSON.stringify({ messages, agent }),
  });

  if (!resp.ok) {
    const data = await resp.json().catch(() => ({}));
    onError(data.error || `Error ${resp.status}`);
    return;
  }

  if (!resp.body) { onError('No response body'); return; }

  const reader = resp.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';
  let streamDone = false;

  while (!streamDone) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });

    let newlineIndex: number;
    while ((newlineIndex = buffer.indexOf('\n')) !== -1) {
      let line = buffer.slice(0, newlineIndex);
      buffer = buffer.slice(newlineIndex + 1);
      if (line.endsWith('\r')) line = line.slice(0, -1);
      if (line.startsWith(':') || line.trim() === '') continue;
      if (!line.startsWith('data: ')) continue;
      const jsonStr = line.slice(6).trim();
      if (jsonStr === '[DONE]') { streamDone = true; break; }
      try {
        const parsed = JSON.parse(jsonStr);
        const content = parsed.choices?.[0]?.delta?.content as string | undefined;
        if (content) onDelta(content);
      } catch {
        buffer = line + '\n' + buffer;
        break;
      }
    }
  }

  // Flush remaining buffer
  if (buffer.trim()) {
    for (let raw of buffer.split('\n')) {
      if (!raw) continue;
      if (raw.endsWith('\r')) raw = raw.slice(0, -1);
      if (raw.startsWith(':') || raw.trim() === '') continue;
      if (!raw.startsWith('data: ')) continue;
      const jsonStr = raw.slice(6).trim();
      if (jsonStr === '[DONE]') continue;
      try {
        const parsed = JSON.parse(jsonStr);
        const content = parsed.choices?.[0]?.delta?.content as string | undefined;
        if (content) onDelta(content);
      } catch { /* ignore */ }
    }
  }

  onDone();
}
```

const GENERATE_URL = `${YOUR_BACKEND_URL}/functions/v1/generate-image`;

export type ImageQuality = 'fast' | 'hd';

interface GenerateResult {
  imageUrl: string;
  description?: string;
  model?: string;
}

export async function generateImage(
  prompt: string, quality: ImageQuality = 'fast', editImageUrl?: string
): Promise<GenerateResult> {
  const resp = await fetch(GENERATE_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${YOUR_API_KEY}`,
    },
    body: JSON.stringify({ prompt, quality, editImageUrl }),
  });
  if (!resp.ok) {
    const data = await resp.json().catch(() => ({}));
    throw new Error(data.error || `Erro ${resp.status}`);
  }
  return await resp.json();
}

BACKEND — EDGE FUNCTIONS
================================================================================

### 1. chat/index.ts (Chat com streaming)

- CORS headers obrigatórios
- Recebe { messages, agent } no body
- Seleciona system prompt baseado no agente (7 prompts especializados + 1 default)
- REGRA IMPORTANTE DOS PROMPTS: Todos incluem a instrução "Nunca se apresente, nunca diga quem você é, nunca faça introduções sobre si mesmo. Vá direto ao ponto."
- Faz fetch para API de completions com stream: true
- Modelo: qualquer modelo compatível OpenAI (ex: gpt-4, gemini-flash, etc.)
- Retorna o body da response como stream (text/event-stream)
- Trata erros 429 (rate limit) e 402 (sem créditos)

System prompts dos agentes (resumo):
- research: pesquisa, dados, resumo, fact-checking
- coder: código, debug, arquitetura, code review
- business: mercado, estratégia, startups, monetização
- marketing: campanhas, copywriting, growth, funis
- content: artigos, roteiros, redes sociais, scripts
- analyst: dados, relatórios, padrões, tendências
- automation: fluxos, APIs, automação
- default: assistente geral com 4 modos (conversacional, execução, análise, automação)

### 2. generate-image/index.ts (Geração de imagens)

- Recebe { prompt, quality, editImageUrl }
- quality "fast" → modelo rápido (ex: gemini-flash-image / dall-e)
- quality "hd" → modelo detalhado (ex: gemini-pro-image)
- Se editImageUrl fornecido, envia como multimodal (text + image_url)
- Usa modalities: ["image", "text"]
- Retorna { imageUrl, description, model }
- A imageUrl vem de: response.choices[0].message.images[0].image_url.url
- Trata erros 429, 402, e quando não gera imagem (422)

COMPORTAMENTOS IMPORTANTES
================================================================================

1. TEMA: Padrão é "dark". Salvo em localStorage key 'nexusia-theme'. Classe 'dark' toggleada no html element.

2. STREAMING: Respostas aparecem token-a-token em tempo real. O conteúdo streaming é mostrado como mensagem temporária (id='streaming') que é substituída pela mensagem final quando onDone é chamado.

3. CONVERSAS: Armazenadas apenas em memória (Zustand). Não há persistência em banco de dados. A primeira mensagem do usuário define o título da conversa (primeiros 40 chars).

4. GERAÇÃO DE IMAGENS: Dois comandos — /imagine (rápido) e /imaginehd (HD). Imagens são exibidas como attachments nas mensagens do assistente com botão de download.

5. SIDEBAR: Animação suave de abertura/fechamento. Quando fechada, botão flutuante no canto superior esquerdo para reabrir.

6. MARKDOWN: Respostas do assistente renderizam markdown completo com syntax highlighting em blocos de código, botão copiar, e imagens.

7. RESPONSIVIDADE: Grid de agentes adapta de 1 a 4 colunas. Sidebar com largura fixa de 280px.

NOTAS FINAIS

- O app NÃO tem autenticação — é aberto
- O app NÃO persiste conversas em banco — tudo em memória
- A interface é inspirada no ChatGPT/Claude com sidebar + chat
- O design é minimalista e moderno com bordas sutis e cores desaturadas
- Animações são suaves e não excessivas
- Todo texto da interface está em português brasileiro
- Os agentes NUNCA devem se apresentar nas respostas (regra nos system prompts)

Para adaptar a outro backend de IA, basta:
1. Substituir as URLs das edge functions (CHAT_URL e GENERATE_URL)
2. Ajustar headers de autenticação
3. Garantir que a API retorne formato compatível com OpenAI (SSE com data: {choices: [{delta: {content}}]})
